# Bundle Format Reference

This document describes the JSON format of bundle manifests generated by `cargo xtask bundle`.

## File Location

Bundles are written to:

```
artifacts/bundles/<sanitized_selector>.json
```

The selector is sanitized by replacing non-alphanumeric characters (except `-` and `_`) with underscores.

## Top-Level Structure

```json
{
  "selector": "AC-XK-SEC-INLINE-001",
  "scenarios": [
    /* Array of ScenarioRecord objects */
  ]
}
```

## Fields

### `selector`
- **Type:** `string`
- **Required:** Yes
- **Description:** The selector string used to create this bundle. Determines which scenarios are included.

### `scenarios`
- **Type:** `array` of `ScenarioRecord`
- **Required:** Yes
- **Description:** All scenarios that matched the selector. May contain one or many scenarios depending on the selector type.

## ScenarioRecord Fields

### `scenario_id`
- **Type:** `string`
- **Required:** Yes
- **Format:** `SCN-XK-<LAYER>-<NNN>`
- **Example:** `SCN-XK-SEC-INLINE-001`
- **Description:** Unique identifier for the scenario. Used in impact analysis and test execution.

### `ac_id`
- **Type:** `string` (nullable)
- **Required:** No
- **Format:** `AC-XK-<LAYER>-<NNN>`
- **Example:** `AC-XK-SEC-INLINE-001`
- **Description:** Acceptance Criteria ID this scenario tests. Multiple scenarios may share the same AC ID.

### `req_id`
- **Type:** `string` (nullable)
- **Required:** No
- **Format:** `REQ-XK-<LAYER>[-<NAME>]`
- **Example:** `REQ-XK-SEC-INLINE`
- **Description:** Requirement ID this scenario belongs to. Groups related ACs and scenarios.

### `feature_file`
- **Type:** `string`
- **Required:** Yes
- **Example:** `specs/features/sec/inline_restrictions.feature`
- **Description:** Path to the Gherkin feature file containing this scenario.

### `sidecar_file`
- **Type:** `string`
- **Required:** Yes
- **Example:** `specs/features/sec/inline_restrictions.meta.yaml`
- **Description:** Path to the YAML metadata sidecar file with scenario configuration.

### `layer`
- **Type:** `string`
- **Required:** Yes
- **Values:** `foundation`, `taxonomy`, `inline`, `sec`, `cli`, `export`, `workflow`
- **Example:** `sec`
- **Description:** Architecture layer this scenario belongs to.

### `module`
- **Type:** `string`
- **Required:** Yes
- **Example:** `FEAT-XK-SEC-INLINE:inline-restrictions`
- **Description:** Module identifier, typically `FEAT-ID:feature-name` format.

### `crates`
- **Type:** `array` of `string`
- **Required:** No (defaults to empty array)
- **Example:** `["ixhtml-scan", "efm-rules", "validation-run"]`
- **Description:** List of crate names this scenario exercises. Used in impact analysis.

### `fixtures`
- **Type:** `array` of `string`
- **Required:** No (defaults to empty array)
- **Example:** `["synthetic/sec/inline/ix-fraction-01"]`
- **Description:** List of fixture paths needed to run this scenario.

### `profile_pack`
- **Type:** `string` (nullable)
- **Required:** No
- **Example:** `sec/efm-77/opco`
- **Description:** Profile pack path for scenarios that require taxonomy/profile data.

### `receipts`
- **Type:** `array` of `string`
- **Required:** No (defaults to empty array)
- **Example:** `["validation.report.v1", "scenario.run.v1"]`
- **Description:** Expected receipt types produced when this scenario runs.

### `allowed_edit_roots`
- **Type:** `array` of `string`
- **Required:** No (defaults to empty array)
- **Example:** `["crates/ixhtml-scan", "specs/features/sec"]`
- **Description:** Paths that can be edited without requiring this scenario to be re-tested. Used in impact analysis.

### `suite`
- **Type:** `string` (nullable)
- **Required:** No
- **Example:** `synthetic`
- **Description:** Test suite this scenario belongs to.

### `speed`
- **Type:** `string` (nullable)
- **Required:** No
- **Values:** `fast`, `medium`, `slow`
- **Example:** `fast`
- **Description:** Execution speed classification for test prioritization.

## Complete Example

```json
{
  "selector": "AC-XK-SEC-INLINE-001",
  "scenarios": [
    {
      "scenario_id": "SCN-XK-SEC-INLINE-001",
      "ac_id": "AC-XK-SEC-INLINE-001",
      "req_id": "REQ-XK-SEC-INLINE",
      "feature_file": "specs/features/sec/inline_restrictions.feature",
      "sidecar_file": "specs/features/sec/inline_restrictions.meta.yaml",
      "layer": "sec",
      "module": "FEAT-XK-SEC-INLINE:inline-restrictions",
      "crates": [
        "ixhtml-scan",
        "ixds-assemble",
        "efm-rules",
        "validation-run",
        "sec-profile-types",
        "xbrl-report-types",
        "scenario-runner",
        "xbrlkit-bdd",
        "xbrlkit-bdd-steps",
        "xbrlkit-cli",
        "xtask"
      ],
      "fixtures": [
        "synthetic/sec/inline/ix-fraction-01"
      ],
      "profile_pack": "sec/efm-77/opco",
      "receipts": [
        "validation.report.v1",
        "scenario.run.v1"
      ],
      "allowed_edit_roots": [
        "crates/ixhtml-scan",
        "crates/ixds-assemble",
        "crates/efm-rules",
        "crates/validation-run",
        "crates/sec-profile-types",
        "crates/xbrl-report-types",
        "crates/scenario-runner",
        "crates/xbrlkit-bdd",
        "crates/xbrlkit-bdd-steps",
        "crates/xbrlkit-cli",
        "xtask",
        "specs/features/sec",
        "profiles/sec/efm-77/opco"
      ],
      "suite": "synthetic",
      "speed": "fast"
    }
  ]
}
```

## JSON Schema

The bundle format is defined in Rust at `crates/scenario-contract/src/lib.rs`:

```rust
#[derive(Debug, Clone, PartialEq, Eq, Serialize, Deserialize, Default)]
pub struct BundleManifest {
    pub selector: String,
    #[serde(default)]
    pub scenarios: Vec<ScenarioRecord>,
}
```
