# Security Policy

`xbrlkit` is an alpha repository. Please do not use public issues for suspected security problems.

## Reporting

If GitHub private vulnerability reporting is enabled for the repository, use that path first.

If it is not enabled, contact the maintainers through a private GitHub channel before disclosing details publicly.

## Scope

Security reports are most useful when they include:

- the affected command or crate
- the fixture or input shape that triggers the problem
- the observed impact
- a minimal reproduction

## Disclosure

Please allow the maintainers time to confirm the issue and prepare a fix before public disclosure.
