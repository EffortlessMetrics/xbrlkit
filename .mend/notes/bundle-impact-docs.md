## Research Summary

### Bundle Command (`cargo xtask bundle <selector>`)

**Location:** `xtask/src/main.rs` - `bundle()` function

**Purpose:** Creates a bounded context bundle for a given selector. The bundle is a JSON manifest containing all scenarios that match the selector.

**Selector Matching Rules:**
- Exact match on `scenario_id` (e.g., `SCN-XK-WORKFLOW-001`)
- Exact match on `ac_id` (e.g., `AC-XK-SEC-INLINE-001`)
- Exact match on `req_id` (e.g., `REQ-XK-WORKFLOW`)
- Tag-style selectors with `@` prefix (e.g., `@SCN-XK-WORKFLOW-001`, `@AC-XK-SEC-INLINE-001`)

**Output:** `artifacts/bundles/<sanitized_selector>.json`

**BundleManifest Structure:**
```rust
struct BundleManifest {
    selector: String,           // The selector that was used
    scenarios: Vec<ScenarioRecord>,  // All matching scenarios
}
```

**ScenarioRecord Fields:**
- `scenario_id`: Unique identifier (e.g., `SCN-XK-SEC-INLINE-001`)
- `ac_id`: Acceptance Criteria ID (e.g., `AC-XK-SEC-INLINE-001`)
- `req_id`: Requirement ID (e.g., `REQ-XK-SEC-INLINE`)
- `feature_file`: Path to Gherkin feature file
- `sidecar_file`: Path to metadata YAML file
- `layer`: Architecture layer (e.g., `sec`, `cli`, `workflow`)
- `module`: Module identifier
- `crates`: List of crates involved in this scenario
- `fixtures`: List of test fixtures needed
- `profile_pack`: Profile pack path (if applicable)
- `receipts`: Expected receipt types
- `allowed_edit_roots`: Paths that can be edited without breaking this scenario
- `suite`: Test suite name (e.g., `synthetic`)
- `speed`: Execution speed tag (e.g., `fast`)

### Impact Command (`cargo xtask impact --changed <paths...>`)

**Location:** `xtask/src/main.rs` - `impact()` function

**Purpose:** Estimates which scenarios are impacted by changes to specific file paths. This helps developers understand which tests need to be run after making changes.

**Impact Detection Logic:**
A scenario is considered impacted if:
1. Any changed path starts with one of the scenario's `allowed_edit_roots`
2. OR the changed path contains one of the scenario's `crates`

**Output:** `artifacts/impact/impact.report.v1.json`

**ImpactReport Structure:**
```rust
struct ImpactReport {
    changed_paths: Vec<String>,       // Normalized paths that changed
    impacted_scenarios: Vec<String>,  // Scenario IDs affected
}
```

**Path Normalization:**
- Converts `\` to `/` (Windows compatibility)
- Strips leading `./` prefixes

### Artifact Directory Structure

```
artifacts/
├── bundles/           # Bundle manifests (*.json)
├── impact/            # Impact reports (impact.report.v1.json)
├── runs/              # Scenario run receipts
├── feature.grid.v1.json   # Compiled feature grid
└── ...
```

### Feature Grid

**Location:** Generated by `cargo xtask feature-grid` → `artifacts/feature.grid.v1.json`

**Purpose:** Compiles all scenarios from Gherkin feature files and their sidecar metadata into a single JSON file. This is the source of truth for bundle and impact analysis.

**Source:** 
- Gherkin `.feature` files under `specs/features/`
- Sidecar `.meta.yaml` files alongside features
